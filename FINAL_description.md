# Итоговое задание по модулю «Интеграция систем и документирование API»

Кинотеатр “Искорка”, для которого мы ранее разработали многофункциональный API, планирует запустить бонусную программу для постоянных посетителей.

## Условия заказчика:

* Участие в бонусной программе необязательно, при желании посетитель может оформить бонусную карту.
* Если клиент указывает при бронировании номер карты, то 10% стоимости бронирования зачисляется на карту в виде бонусных баллов (1 балл = 1 рубль).
* Баллы начисляются автоматически сразу после оплаты брони.
* При бронировании клиент может оплачивать до 50% стоимости билетов бонусными баллами.
* Также руководство кинотеатра ставит задачу - представлять ежемесячный отчет:
  - как часто клиенты используют бонусную карту при бронировании
  - какую часть стоимости билетов (%) клиенты оплачивают бонусными баллами

## Решение
### Какие изменения потребуется внести в существующие структуры данных для решения задачи?

1. Потребуется расширить существующую таблицу **Бронирования / booking** (добавляем информацию по бронированию в части оплаты бонусами)

|Данные|Обозначение|Тип данных|
|-|--------|---|
|Идентификатор|bookingId|integer|
|Пользователь|customerId|integer|
|Сеанс|sessionId|integer|
|Выбранное место: Ряд (1-7), Место (1-9)|place: row; seat|object: integer, enum (1-7); integer, enum (1-9)|
|Статус: новый, завершен, удален|status: active; done; canceled|string, enum: active; done; canceled|
|Использовалась ли бонусная карта|usedBonusCard|boolean|
|Сумма (в рублях), оплаченная бонусными баллами, максимум 50% от цены|bonusPaymentAmount|string|
|Сумма, оплаченная реальными деньгами|cashPaymentAmount|string|
|Баллы, начисленные после оплаты (10% от полной стоимости)|earnedBonus|string|

### Требуется ли добавление новых таблиц для решения задачи? Если да, то каких? 

1. Новая таблица реестр бонусных карт **Бонусные карты / bonus_cards**

|Данные|Обозначение|Тип данных|
|-|--------|---|
|Идентификатор|cardId|integer|
|Номер бонусной карты|bonusCardNumber|string (уникальный, может быть null, формат хххх-хххх-хххх-хххх)|
|Текущий баланс бонусных баллов|bonusBalance|integer|
|Пользователь|customerId|ID пользователя FK|

2. Новая таблица для хранения транзакций с бонусными баллами **история операций с бонусами / bonus_transaction**

|Данные|Обозначение|Тип данных|
|-|--------|---|
|Идентификатор|transactionId|integer|
|Пользователь|customerId|ID пользователя FK|
|Бронирование|bookingId|ID бронирования FK|
|Тип тразакции (списание, начисление)|transactionType|string, enum: earn; spend|
|Сумма баллов|amount|string|
|Дата и время транзакции|createdAt|string (date-time)|

### Какие изменения потребуется внести в существующие методы API для решения задачи?

1. Метод **POST /booking** (создание заказа): потребуется добавить новые параметры при создании (дополнить тело запроса).
2. Метод **PATCH / booking/{bookingId}**: при переходе в status=done проверять оплату и начислять бонусы если не начислено.
3. Метод **GET /booking**: добавить фильтр usedBonusCard (true/false) для отчета.
4. Дополнить блок **components** для схемы ответа по booking.

### Требуется ли создание новых методов? Если да, опишите новые методы

1. Оформление бонусной карты
* **POST/bonus_cards**
* Параметры запроса: customerId (integer, required)
* Параметры ответа: card_id (integer), bonusBalance

2. Получение баланса бонусной карты
* **GET/bonus_cards/{cardId}**
* Параметры запроса: cardId (integer)
* Параметры ответа: card_id (integer), bonusCardNumber(string), bonusBalance (integer), customer_id (integer)

3. История транзакций карты
* **GET/bonus_cards/{cardId}/transactions**
* Параметры запроса: cardId (integer), page (query, int=1), size (query, int=20), type (query, enum=credit/debit/all)
* Параметры ответа: items (array: transactionId, transactionType, amount, createdAt), pagination

4. Ежемесячный отчет по бонусам
* **GET/reports**
* Параметры запроса: month (query, string=YYYY-MM, required)
* Параметры ответа: totalBookingsWithCard (int, брони с картой), totalBookings (int, все брони), cardUsagePercent (string, % брони с картой), avgBonusPaidPercent (string, ср. % оплаты баллами от totalAmount), totalBonusesCredited (string), totalBonusesUsed (string)
